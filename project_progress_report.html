运行流程

1. 订阅维护

用户在 WeWe RSS dashboard 手动增删公众号。

2. 自动触发

Codex 自动化早盘/收盘执行 run-once。

3. 全文导入

读取 /feeds 和单号 mode=fulltext JSON。

4. 入库去重

SQLite 按 URL 去重，识别本轮新增文章。

5. 摘要看板

生成本地摘要，刷新 output/index.html。

6. 日报归档

有新增文章时生成日报 HTML/JSON 和索引。

已知问题与后续路线

  1. 深度总结尚未接模型 API。 当前摘要是本地抽取式，日报只是基于已有摘要做归纳。下一步应接入模型，按“核心观点、关键证据、市场含义、风险提示、完整性说明”生成高质量总结。
  2. 综合号分类偏粗。 广发、华泰等综合号下包含宏观、策略、金工、固收、行业等内容，目前主要按订阅源归类，后续可用标题/正文关键词二级分类。
  3. 新增公众号仍需人工。 WeWe 远端转换接口可能被 Cloudflare 拦截，稳定做法仍是用户在 dashboard 手动添加订阅。
  4. WeWe RSS 依赖账号状态。 如果 WeRead 账号被标记失效，需要在 dashboard 账号管理里启用，或使用 skill 中的 enable-accounts --latest 辅助恢复本地状态。
  5. 大摩 Robin Xing / Laura Wang 暂缺。 Morgan Stanley 公开网页、PDF、邮件或授权平台未纳入当前自动化，后续需要单独开发外资行来源模块。
  6. 历史日报策略待定。 当前策略是每次有新增文章就生成一份本轮日报；如果未来需要“每日唯一日报”，可改为同日追加/覆盖聚合版日报。

下一位接手者建议

  1. 先运行 python -m src.cli status，确认 WeWe RSS 服务和数据库状态。
  2. 打开 output/index.html 检查主看板筛选、排序和日报入口。
  3. 打开 output/reports/index.html 检查日报索引。
  4. 如果要验证日报，不要先删库；可以运行 python -m src.cli daily-report 手动生成当天测试日报。
  5. 如果要开发深度总结，优先在 src/reports/daily.py 和 src/summarizer/ 下新增模型总结模块，避免破坏现有本地摘要。
