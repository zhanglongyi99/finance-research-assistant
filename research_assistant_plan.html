AI研报助手开发执行计划

目标：先把免费/已授权可见内容的采集、归档、AI总结和本地网页看板跑通；微信小程序 GUI 自动化放到后续阶段。更新时间：2026-05-19。

1. 项目目标

开发一个本地运行的 AI 研报助手，自动收集微信公众号、券商网页、公开 PDF、财经媒体转载等内容，保存原文入口和本地 PDF，生成 AI 总结，并汇总到一个本地网页端。

核心策略：第一版先做稳定的数据闭环，不把券商微信小程序作为主线。小程序后续采用微信 PC GUI 自动化，尽量减少人工操作，但等主要采集链路稳定后再接入。

2. 内容优先级

优先级 内容来源 处理方式 第一版状态
P0 微信公众号 第三方公众号解析 API + 链接解析；失败时记录待补正文。 立即实现
P0 券商网页/研究所网页 网页抓取、搜索模板、公开 PDF 下载。 立即实现
P1 东方财富、慧博公开页、财经媒体转载 抓标题、摘要、日期、链接；发现公开 PDF 则下载。 第一版补充源
P2 券商微信小程序 后续用微信 PC GUI 自动化；先在数据模型中预留字段。 暂缓开发
不做 付费墙、私有接口逆向、绕过验证码 不进入第一版范围。 排除

5. 系统模块

模块 职责 建议实现
采集器 公众号 API、网页抓取、搜索结果解析、PDF 下载。 Python 脚本；每个来源一个 adapter。
入库与去重 按 URL、标题、PDF 哈希去重，记录采集状态。 第一版用 JSONL 或 SQLite；推荐 SQLite。
文本提取 从网页正文和 PDF 中提取可总结文本。 网页用解析器；PDF 用文本提取库。
AI 总结 生成固定格式总结，并标注内容是否完整。 调用模型 API；失败时保留待总结状态。
网页看板 展示新增内容、PDF、总结、缺口和人工待处理项。 静态 HTML + JSON 数据即可。
自动化调度 定时运行采集与更新任务。 Codex 自动化或 Windows 任务计划程序。

9. AI总结格式

每篇内容统一生成这四块，方便横向比较。

核心观点：
- 用 2-4 条列出作者最重要的判断。

关键证据/数据：
- 摘出关键宏观数据、市场数据、行业指标或政策依据。

市场含义：
- 说明对股票、债券、汇率、商品或行业配置的可能影响。

完整性说明：
- 标明总结基于完整原文、公开摘要、网页转载，还是仅标题线索。

10. 执行路线

第 1 步
建立新项目目录、配置文件、SQLite 数据库和基础命令入口。
第 2 步
跑通 3 个公众号、3 个关键词、2 个网页源的最小采集闭环。
第 3 步
加入 PDF 下载、文本提取、去重和失败状态记录。
第 4 步
接入 AI 总结，输出固定格式摘要。
第 5 步
生成本地网页看板，展示新增内容、总结、PDF 和待处理缺口。
第 6 步
连续试运行两周，统计缺口后再做券商小程序 GUI 自动化。

11. 推荐命令入口

新项目可设计成以下命令。具体命令名可调整，但能力边界建议保持清晰。

python -m src.cli init
python -m src.cli collect --source wechat
python -m src.cli collect --source web
python -m src.cli summarize --pending
python -m src.cli render-dashboard
python -m src.cli run-once

run-once 用于 Codex 自动化或 Windows 任务计划程序定时触发：采集、入库、总结、渲染网页一次跑完。

12. 测试与验收

  • 用 3 个公众号、3 个关键词、2 个公开网页源先做最小闭环。
  • 重复运行两次，确认同一 URL、标题和 PDF 不会重复入库。
  • 确认公开 PDF 能下载、提取文本、生成总结。
  • 确认本地网页能正确展示中文、链接、PDF 路径、AI 总结和状态。
  • 确认失败内容不会中断流程，而是进入 need_manual 或 summary_pending。
  • 两周后统计缺口，再决定小程序 GUI 自动化优先级。

13. 后续阶段

  1. 先跑通公众号、网页和公开 PDF。
  2. 连续试运行两周，统计哪些核心内容缺失。
  3. 再为广发、华泰、兴业、华创、华西等券商小程序设计微信 PC GUI 自动化。
  4. 最后再考虑 HERMES 或其他本机助手作为通知、调度或对话入口。
