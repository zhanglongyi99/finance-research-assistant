产品定位

第一版内核

  • 以研报阅读助手为主，不直接开发泛财经全能助手。
  • 首屏围绕今日/本轮新增研报的 AI 简报。
  • 核心交互是基于研报库的引用式问答。
  • 默认只基于已入库研报回答，证据不足时明确说明。

长期方向

  • 逐步演进为个人财经研究中枢。
  • 未来接入实时金融数据、财经新闻、盘中提醒和专题研究。
  • 行情、新闻、提醒在第一版 UI 中只做灰度占位。
  • 联网搜索仅在用户明确要求时启用。

问题与风险

  1. 开源隐私边界：数据库、原始 HTML、输出内容、日志、API key、账号状态和本地路径需要从 Git 仓库中排除或模板化。
  2. WeWe RSS 依赖：当前采集链路依赖本地 WeWe RSS 服务和账号状态，云端化时需要重新设计部署方式。
  3. 摘要质量：当前是本地抽取式摘要，容易混入作者信息、免责声明和排版噪声，需要模型总结和正文清洗升级。
  4. 引用可信度：问答必须保持证据边界，库内材料不足时不能编造；第一版先做到文章级引用。
  5. 云端部署：当前 SQLite、静态 HTML、Codex 自动化都偏本地，云端化需要拆分后端、前端、数据库和调度。
  6. 外资行和网页来源：Morgan Stanley 等来源尚未纳入当前生产主线，后续要单独确认授权和公开来源策略。
  7. 测试覆盖：当前没有正式测试目录，后续功能开发需要补充最小自动化验证。
